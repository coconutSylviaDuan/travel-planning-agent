# 海外目的地调研工作流（Google Maps + 小红书）

## 适用场景

目的地在中国大陆以外（港澳台、日韩、东南亚、欧美等），大众点评覆盖不足或不可用。

## 双源策略

| 数据源 | 用途 | 覆盖范围 |
|--------|------|----------|
| **小红书** | 中文用户的真实行程攻略、氛围体验、避坑提醒、近期实况 | 全球（华人旅行者聚集） |
| **Google Maps** | 餐厅评分/评价量/价格/营业时间/热门时段/实景照片、路线导航 | 全球（覆盖最完整） |

## Phase A：小红书行程调研

与国内版小红书调研完全一致，走 `references/xhs-research.md` 的 CDP 流程。

海外搜索词模板：

```bash
# 按城市+天数搜行程
"东京 3天2夜 行程"
"巴黎 4天 自由行 攻略"
"曼谷 美食 路线"

# 按区域搜餐厅
"新宿 午餐 推荐"
"左岸 咖啡馆"
"弘大 烤肉"

# 按需求搜
"带爸妈 京都 轻松路线"
"巴厘岛 避坑"
"冰岛 自驾 冬季"
```

海外小红书内容的特殊筛选规则：

### 保留（真实信号）
- 明确写了日期（看出季节和时效性）
- 有具体价格（当地货币/人民币换算）
- 提到交通方式（地铁线路、步行距离）
- 有排队/预约体验
- 涉及签证、支付、语言等实用信息

### 不保留
- 纯拍照打卡合集、无实际体验
- "必去"类泛泛清单
- 旅行社广告
- 明显搬运国内游记

## Phase B：Google Maps 餐厅验证

### 搜索方式

直接使用 Google Maps 网页版或 API 搜索：

```
搜索词格式：
"restaurants near {地标/区域} {城市}"
"{区域} lunch"
"{菜系} near {酒店名}"
```

### 判断维度（替代大众点评的硬信号）

| 维度 | Google Maps 对应字段 | 判断标准 |
|------|---------------------|----------|
| 口味稳定性 | `rating` (1-5) | ≥4.0 可信，≥4.3 优秀 |
| 评价量 | `user_ratings_total` | ≥100 信号可靠，<50 样本太小 |
| 价格 | `price_level` (1-4) | 匹配预算 |
| 营业时间 | `opening_hours` | 确认当天是否营业、午休时间 |
| 热门时段 | `popular_times` | 避开高峰（排队风险） |
| 实景 | Photos tab | 看环境、菜单、份量 |
| 近期评价 | Reviews → Sort by Newest | 看最近1-3个月的反馈 |
| 菜系标签 | `types` | 确认菜系是否符合这顿饭的需求 |

### 评价分析要点

Google Maps 评价以英文为主，看的时候关注：

- **排队**：review 里出现 "wait"/"queue"/"line"/"reservation"
- **踩雷**：低分里反复出现同一问题（服务差、不新鲜、价格虚高）
- **游客店**：review 语言分布全是非本地语言 + 评分虚高但评价少
- **本地人推荐**：review 里有本地语言（日文/韩文/泰文等）且评分高
- **性价比**：price_level + review 里 "$"/"expensive"/"worth it"

### 写回格式

每顿饭保留 2-3 个候选：

```md
午餐区域：银座 / 有乐町（东京）

主推：店名 A（Google Maps 4.3 · ¥2,500-4,000 · 1200+ reviews）
- GMaps：评分稳定，评价量大，本地人+游客混合好评，午市套餐性价比高
- 小红书：近期多篇笔记推荐，排队约15分钟，适合工作日午餐

备选：店名 B（Google Maps 4.1 · ¥1,500-2,500 · 800+ reviews）
- GMaps：离银座站步行3分钟，翻桌快，适合赶时间
- 小红书：更像是简餐，氛围一般但口味稳定
```

## Phase C：Google Maps 导航链接

在行程地图的导航按钮中，海外目的地使用以下优先级：

1. **Google Maps** — 全球最完整，直接拼接 URL：
   ```
   https://www.google.com/maps/dir/?api=1&destination={lat},{lng}&travelmode={mode}
   ```
   - `travelmode`：`driving` / `walking` / `transit`

2. **Apple Maps**（仅 iOS 用户）— 作为备选：
   ```
   https://maps.apple.com/?daddr={lat},{lng}&dirflg={mode}
   ```

3. **不推荐高德/百度地图** — 海外数据覆盖不足

## 支付与语言提示

海外行程需额外标注：

- **支付方式**：当地主流支付（信用卡、现金、当地电子钱包）+ 支付宝/微信覆盖情况
- **语言**：当地通用语言、英语可用程度、是否需要翻译工具
- **签证/入境**：是否需要签证、入境注意事项

在生成地图页面的 `overviewContent()` 中增加"出行须知"区域。

## 常见坑

- 用大众点评查海外餐厅 → 数据稀疏，信号不可靠
- 只用小红书种草，不用 Google Maps 验证 → 踩网红店雷
- 忽略当地用餐时间差异（如西班牙晚餐 9pm 才开始）
- 忽略小费文化（美国 15-20%）
- 忽略餐厅午休（日本很多餐厅 14:00-17:00 不营业）
- 忽略预约文化（日本高级餐厅必须预约，walk-in 基本没戏）
